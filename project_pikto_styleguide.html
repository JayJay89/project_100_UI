  • Introduction of Foundational Components

    Colors, Shadows, Border-radii, Spacings, Typography, Transitions, Animations

    These are the basic modular components used in the constructions of complex elements such as buttons, accordions, labels, etc. Each complex element will be formed from the variables and mixins defined for the foundational components

    The general idea is that having a set of foundational components which follows well-defined rules. Our complex components can be composed from foundational components, and would the overall result is a much more cohesive system. Modularization is the key attribute in the project.

  • Splitting mixins and variables into chunks to reflect this change of management

    Originally we had variables.scss, and mixins.scss, to accomodate new changes of this management. This is a work in progress, typography and transitions isn't modularized yet

    @import "variables-colors"; @import "variables-spacings"; @import "variables"; @import "mixins-border-radii"; @import "mixins-color-combo"; @import "mixins-shadow"; @import "mixins-typography"; @import "mixins";
    code

  • Bootstrap Code - Keep it lean

    Bootstrap provides us with many useful components that helps us in development. However, at the same time it bloats our code, by keeping a lot of components that we didn't make use of. Aside from that, using Bootstrap prevents us from fully implementing our design system, which is constructing our system from modular fundamental components.


    The actionable step to be taken here - is to comb through our current code base, keep what we need, and dispose codes that we don't. This means slowly converting our design system for our own need. In that process, we could learn how bootstrap implement certain elements and how we could benefit from that.

    If bootstrap 4 comes up a feature that could be of use to use. We could extract the code that we need and adapt it to our arsenal.

  • Variables with one case usages are removed

    We have multiple variables which are only used once. Example: $alert-border-radius is only used to defined the border-radius for alerts. This could be easily expressed without using variables as well. Other than that, to change that value, we had to search for $alert-border-radius

  • Constructor Mixins and Shared Mixins

    There are two kind of mixins, There is a constructor and a shared mixin

    Constructor Mixins

    They are only used to generate properties for an element from the given inputs.

    Example: alert-constructor, button-constructor and label-constructor...etc

    Constructors are usually placed within their related scss. For example, button-constructor are placed within buttons.scss

    Shared Mixins

    Shared Mixins are those which are shared and used by multiple elements.

    Example: clearfix, transform, opacity...etc

    These are all located in mixins.csss

  • Variables

    For Variable linking the max variable links will be: 4

Generating Elements with color combo sets

Teal Link Magenta Link

Here's the color-combo sets

$teal-combo: ( 'col-1': $teal, 'col-2': $teal-dark, 'col-3': $teal-darker ); $magenta-combo: ( 'col-1': $magenta, 'col-2': $magenta-dark, 'col-3': $magenta-darker );
code

This is the Link Generator, the factory

@mixin generate-link($combo){ color: map-get($combo, 'col-1'); &:hover { color: map-get($combo, 'col-2'); } &:active { color: map-get($combo, 'col-3'); } }
code

Passing the color combo into the constructor to generate the colors

.general-link { @include generate-link($teal-combo); } .attention-link { @include generate-link($magenta-combo); }
code

In CSS this'll be displayed

.general-link { color: map-get($teal-combo, 'col-1'); &:hover { color: map-get($teal-combo, 'col-2'); } &:active { color: map-get($teal-combo, 'col-3'); } } .attention-link { color: map-get($magenta-combo, 'col-1'); &:hover { color: map-get($magenta-combo, 'col-2'); } &:active { color: map-get($magenta-combo, 'col-3'); } }
code

Spacing

Spacing is the general spacing rules that governs the webapp. This is an effort to keep the spacing and dimension consistent. However, do note that not all elements will be applied to this rule. All spacing configurations are controlled in variables-spacings.scss

$base-size: 4; $s1: $base-size * 1; $s2: $base-size * 2; $s3: $base-size * 3; $s4: $base-size * 4; $s5: $base-size * 5; $s6: $base-size * 6;
code

The base size is 4px, and it increments so on...

.pikto-btn { padding: $s1 $s3; } /* this produces a padding: 4px 12px; */
code

However, this system is not implemented yet as it'll cause a huge drastic changes in the the element's and layout's spacing and sizes. Is there a need for this system or not depends on how important it is to have this kind of system.

On Tags and Classes

For our typography, HTML tags denotes the hierarchy of the headers. As an example <h1> tag are use for the main header, and <h2> is for secondary headers. They are used semantically rather than as a mean to define the font-size.

So how do we define our font sizes units? We use classes instead.

.h1 is the largest font-size and .h6 is the smallest one

Line Height

Min: 20px

In the context of an enterprise application, the need for longform text is quite rare. More often, application design involves many small bits of text that all represent the users data. This text is spaced intentionally to maintain distinction between different types and rows of data.

As a result, we take a pragmatic approach to spacing and line height by resetting all margins and paddings to 0, and allowing the line-height property for all text to inherit a globally defined ratio of 1.5 of the text size. This allows designers and developers to easily opt-in to specific text spacing by leveraging our CSS Text and Spacing Utilities.
